## 重构底层同步方法

更高效、更快速、信息更细、容错更高、操作更细致、资源占用更低，并提升实时性。

* 重做移动模式 [#113](https://github.com/dr34m-cn/taosync/issues/113) [#55](https://github.com/dr34m-cn/taosync/issues/55)
* 作业详情展示进一步细化，可查看总体进度、同步速度、预估剩余时间、预计完成时间等信息，并支持更高频率刷新 [#110](https://github.com/dr34m-cn/taosync/issues/110) [#95](https://github.com/dr34m-cn/taosync/issues/95) [#72](https://github.com/dr34m-cn/taosync/issues/72)
* 细化到目录级别的操作展示 [#96](https://github.com/dr34m-cn/taosync/issues/96) [#34](https://github.com/dr34m-cn/taosync/issues/34)
* 扫描失败不再直接中断任务，失败信息会记录到详情中对应目录 [#91](https://github.com/dr34m-cn/taosync/issues/91) [#78](https://github.com/dr34m-cn/taosync/issues/78)
* 分批向 OpenList 提交任务，降低远端负载并减少异常概率 [#87](https://github.com/dr34m-cn/taosync/issues/87) [#71](https://github.com/dr34m-cn/taosync/issues/71) [#37](https://github.com/dr34m-cn/taosync/issues/37) [#31](https://github.com/dr34m-cn/taosync/issues/31) [#25](https://github.com/dr34m-cn/taosync/issues/25) [#19](https://github.com/dr34m-cn/taosync/issues/19)
* 移除速度选项，改为缓存与扫描间隔配置，便于更细粒度控制
* 其他若干优化与修复
* 支持 windows-arm 平台（实验性）
